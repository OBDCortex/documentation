Embedded Linux Engineer

Core Infrastructure Development

"Establish the foundation. Design and develop the custom Linux OS to create a stable, efficient platform for all system components."

Core Responsibilities ▶
  • Board/BSP & Toolchain: Research and select the optimal SBC. Set up the Board Support Package and cross-compilation toolchain.
  • Build System (Yocto/Buildroot): Manage "recipes" to define all software components in the final Linux image, ensuring it is minimized.
  • Kernel Configuration: Modify the Linux kernel to enable specific drivers (CAN, Ethernet) and disable unused features.
  • Device Tree Integration: Write/modify device tree source (.dts) files to inform the kernel about the board's specific hardware layout.
  • Image Creation & CI: Automate the OS build process using a CI pipeline (e.g., GitHub Actions) for reproducible builds.
Key Deliverables ▶
  • A custom, bootable Linux image (.img file)
  • Software Development Kit (SDK) for other team members
  • A fully automated build pipeline
  • Documentation on how to flash the device
Cross-functional Collaboration ▶
  • Hardware Engineer: For device tree and kernel driver requirements.
  • Software Engineer: To provide the SDK and required system libraries.

Leadership Responsibility

Project Manager & DevOps Lead

Embedded Software Engineer

Application & System Logic

"Implement the core intelligence. Develop the application logic that transforms hardware capabilities into functional, valuable systems."

Core Responsibilities ▶
  • Network Configuration: Write scripts for `systemd-networkd` or `NetworkManager` to handle all device network connections.
  • Data Acquisition Daemon: Write the main background service that continuously reads from the CAN bus, processes data, and logs/publishes it.
  • Systemd Service Management: Create `systemd` unit files to manage the application's lifecycle (auto-start, auto-restart).
  • API & Protocol Design: Implement the communication protocol (REST/MQTT) and define message schemas (JSON/Protobuf) for the mobile app.
  • Logging and Metrics: Implement robust logging using `journald` to track application behavior and diagnose issues.
Key Deliverables ▶
  • The main data acquisition application binary
  • A clearly documented API for the mobile developer
  • Systemd service files
  • Cloud server setup for AI processing
Cross-functional Collaboration ▶
  • Hardware Engineer: To use the HAL for accessing CAN data.
  • Mobile Developer: To define and test the API.
  • Linux Engineer: To get the application included in the OS build.

Leadership Responsibility

Cloud Integration & Communication Lead

Hardware Integration Engineer

Physical Interface Development

"Bridge hardware and software. Create the critical connection between physical components and digital systems through robust interfaces and reliable drivers."

Core Responsibilities ▶
  • Low-Level Hardware Enablement: Define device tree overlays, specifying pin configurations (pinmux) and enabling CAN/Ethernet controllers.
  • SocketCAN Configuration: Use standard Linux tools (`ip`, `cansend`) to configure the CAN interface, setting parameters like bitrate.
  • Hardware Abstraction Layer (HAL): Develop a C/C++ library that provides a simple API for the Software Engineer to access hardware features.
  • Diagnostics & Measurement: Use tools like logic analyzers, `vcan` interfaces, and test scripts to verify hardware functionality and performance.
Key Deliverables ▶
  • A stable and well-documented C/C++ HAL library
  • Device tree source files and overlays
  • Test scripts for hardware validation
  • Primary technical project documentation
Cross-functional Collaboration ▶
  • Linux Engineer: To integrate drivers and device trees into the kernel.
  • Software Engineer: To provide and support the HAL.

Leadership Responsibility

Primary Documentation Lead

Mobile Application Developer

User Interface & Experience

"Create the user experience. Transform technical data into an intuitive, elegant interface that makes complex technology accessible to users."

Core Responsibilities ▶
  • UI/UX Development: Build the Android app's user interface using modern tools like Jetpack Compose.
  • Networking Client: Implement the client-side logic to communicate with the device's API using libraries like Retrofit or Ktor.
  • Device Discovery & Config: Implement a mechanism (mDNS) for the app to find the device on the network and build the UI for initial setup.
  • Data Visualization: Create charts, graphs, and gauges within the app to display diagnostic data clearly.
  • Beta Distribution & Bug Reporting: Manage app distribution for testing via Google Play and integrate a bug reporting system.
Key Deliverables ▶
  • The final Android application package (.apk/.aab)
  • Source code for the Android app
  • User-facing documentation or help guides
  • API documentation from a consumer's perspective
Cross-functional Collaboration ▶
  • Software Engineer: To understand, test, and provide feedback on the device API.
